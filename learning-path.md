# Learning Path Generator

Create structured, personalized curricula with dependency mapping, resource recommendations, and milestone checkpoints.

## Approach

### Phase 1: Learner Assessment
Before designing the path, understand the learner:

1. **Current State**
   - What do they already know?
   - What adjacent knowledge do they have?
   - What's their learning style (visual, hands-on, reading)?
   - How much time can they dedicate?

2. **Target State**
   - What does "learned" mean for them?
   - Practical application or theoretical understanding?
   - What will they DO with this knowledge?
   - What level: awareness, working knowledge, or mastery?

3. **Constraints**
   - Available time (hours/week, total duration)
   - Budget for resources
   - Access to tools/environments
   - Learning context (work, personal, academic)

4. **Motivation & Stakes**
   - Why do they want to learn this?
   - What happens if they don't?
   - Intrinsic vs extrinsic motivation?
   - How urgent is this?

### Phase 2: Knowledge Mapping
Map the learning domain:

1. **Core Concepts**: Essential building blocks
2. **Prerequisites**: What must be known first
3. **Dependencies**: What unlocks what
4. **Rabbit Holes**: Easy to get lost in, scope carefully
5. **Common Misconceptions**: What trips people up
6. **Practice Domains**: Where to apply learning

### Phase 3: Path Architecture
Design the learning journey:

**Levels of Learning**
1. **Exposure**: Heard of it, basic awareness
2. **Comprehension**: Understand how it works
3. **Application**: Can use it in guided contexts
4. **Analysis**: Can solve novel problems with it
5. **Synthesis**: Can combine with other knowledge
6. **Mastery**: Can teach it to others

**Path Structure**
- **Foundation Phase**: Prerequisites and core concepts
- **Core Learning Phase**: Main content and skills
- **Practice Phase**: Application and reinforcement
- **Integration Phase**: Connecting to real work
- **Deepening Phase**: Advanced topics and mastery

### Phase 4: Milestone Design
Create checkpoints to measure progress:

**Milestone Types**
- **Knowledge Check**: Can explain concept correctly
- **Skill Check**: Can perform task without guidance
- **Project Check**: Can complete meaningful project
- **Teaching Check**: Can explain to someone else

**Milestone Criteria**
- Specific and measurable
- Achievable within phase timeframe
- Validatable (can confirm completion)
- Motivating (feels like progress)

### Phase 5: Resource Curation
Select learning resources:

**Resource Types**
- **Structured Courses**: Linear, comprehensive, hand-held
- **Documentation**: Official, authoritative, reference
- **Tutorials**: Step-by-step, practical, focused
- **Books**: Deep, thorough, portable
- **Videos**: Visual, demonstrated, passive
- **Practice Platforms**: Interactive, hands-on, feedback
- **Projects**: Integrative, real-world, portfolio-building
- **Communities**: Support, answers, motivation

**Selection Criteria**
- Quality (well-reviewed, maintained, accurate)
- Fit (matches learning style and level)
- Accessibility (cost, availability, format)
- Currency (up-to-date, still relevant)

### Phase 6: Path Personalization
Tailor to the individual:

1. Adjust pace based on available time
2. Emphasize resources matching learning style
3. Connect to their specific goals
4. Remove irrelevant advanced topics
5. Add relevant specialized topics
6. Include motivational checkpoints

## Quality Checks

Before delivering path, verify:

### Path Quality
- [ ] Dependencies correctly ordered
- [ ] No circular dependencies
- [ ] Scope appropriate to timeframe
- [ ] Milestones at each phase

### Resource Quality
- [ ] Resources are current and available
- [ ] Mix of resource types
- [ ] Free/paid options where applicable
- [ ] Quality verified (reviews, reputation)

### Personalization Quality
- [ ] Matches stated learning style
- [ ] Connects to stated goals
- [ ] Fits time constraints
- [ ] Appropriate difficulty progression

### Completeness
- [ ] Prerequisites identified
- [ ] Common pitfalls noted
- [ ] Success criteria clear
- [ ] Next steps after completion

## Anti-Patterns to Avoid

1. **Tutorial Hell**: All learning, no building
2. **Scope Creep**: Trying to learn everything
3. **Resource Hoarding**: Collecting without doing
4. **Perfectionism**: Must master before moving on
5. **Isolation**: Learning without community/feedback
6. **Passive Learning**: Watching without doing
7. **No Checkpoints**: Can't measure progress
8. **Outdated Resources**: Learning deprecated approaches

## Special Handling

### Technical Skills
1. Emphasize hands-on practice
2. Include real environment setup
3. Build projects that demonstrate competence
4. Connect to job requirements if relevant

### Conceptual/Theoretical Topics
1. Include application examples
2. Connect to what they already know
3. Use multiple explanation approaches
4. Include discussion/teaching milestones

### Fast-Changing Fields
1. Favor official documentation over courses
2. Emphasize learning how to learn
3. Include staying-current strategies
4. Note what fundamentals don't change

### Career Transitions
1. Map to job requirements
2. Include portfolio building
3. Add interview preparation
4. Connect to networking opportunities

## Invocation Examples

User: "I want to learn Python for data analysis"
→ Assess current programming knowledge, target (job, hobby, specific project), time available

User: "How do I get started with machine learning?"
→ Determine math background, coding experience, goal (understand vs build), commitment level

User: "I need to understand financial modeling for my new role"
→ Assess current finance knowledge, specific modeling needs, timeline pressure, available resources
