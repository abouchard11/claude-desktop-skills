# Structured Brainstorm

Transform unstructured ideation into systematic exploration producing categorized, evaluated, and actionable ideas.

## Approach

### Phase 1: Problem Definition
Before generating ideas, define the target:

1. **Core Challenge**: What are we actually solving for?
2. **Success Criteria**: What makes an idea "good"?
3. **Constraints**: What's off the table? (budget, time, values, technical)
4. **Stakeholders**: Who cares about this? What do they need?
5. **Current State**: What's been tried? What's status quo?

### Phase 2: Divergent Exploration
Generate ideas using multiple lenses:

**Conventional Ideas** (Within existing paradigms)
- What do others in this situation typically do?
- What's the standard industry approach?
- What's the safe, proven option?
- What would the textbook say?

**Unconventional Ideas** (Challenge assumptions)
- What if we did the opposite?
- What would a naive outsider try?
- What if our biggest constraint disappeared?
- What's considered "impossible" but actually isn't?

**Combinatorial Ideas** (Remix and blend)
- What if we combined X approach with Y?
- What can we borrow from adjacent domains?
- What two half-solutions could combine?
- What's a hybrid of conventional and wild?

**Constraint-Flipping Ideas** (Use weaknesses)
- How can we turn our constraint into an advantage?
- What would we do if we had to use our weakness?
- What's the opportunity inside the obstacle?

**Wild Ideas** (No limits)
- If money were no object?
- If we had magic/perfect technology?
- If we could start from scratch?
- What would be hilarious but might work?

### Phase 3: Idea Forcing Techniques
Systematically push beyond obvious:

**Attribute Listing**
- List all attributes of the problem/solution
- Generate variations for each attribute
- Combine variations in new ways

**Analogical Thinking**
- How does nature solve this?
- How do other industries handle this?
- What would [famous person/company] do?
- What's a parallel problem with known solutions?

**Reversal**
- What if we wanted the opposite outcome?
- What if users were our competitors?
- What if we had to prevent success?
- Work backward from the solution

**Random Stimulus**
- Pick random word and force connection
- Find inspiration in unrelated domain
- Constraint injection (add arbitrary limitation)

**SCAMPER Framework**
- Substitute: What can be replaced?
- Combine: What can merge?
- Adapt: What can be borrowed?
- Modify: What can change scale/form?
- Put to other uses: What else could this do?
- Eliminate: What can be removed?
- Reverse: What can be flipped/reordered?

### Phase 4: Idea Clustering
Organize ideas into meaningful groups:

**Clustering Approaches**
- By theme/approach type
- By resource requirements
- By risk level
- By time to implement
- By stakeholder impact
- By innovation level (incremental → breakthrough)

### Phase 5: Idea Evaluation
Assess ideas on relevant dimensions:

**Evaluation Criteria** (Select relevant ones)
- Impact: How much does this solve the problem?
- Feasibility: Can we actually do this?
- Cost: What resources are required?
- Time: How long to implement?
- Risk: What could go wrong?
- Alignment: Does this fit our values/strategy?
- Novelty: How differentiated is this?
- Reversibility: Can we undo if wrong?

**Quick Sort Method**
- Must Do: High impact, high feasibility
- Should Do: High impact, lower feasibility (invest to enable)
- Could Do: Lower impact, high feasibility (quick wins)
- Won't Do: Low impact, low feasibility (eliminate)

### Phase 6: Idea Development
For promising ideas, add detail:

1. **Concrete Next Step**: What would we do first?
2. **Key Assumption**: What must be true for this to work?
3. **Quick Test**: How could we validate cheaply?
4. **Blocker**: What would stop this?
5. **Champion**: Who would drive this?

## Quality Checks

Before delivering brainstorm, verify:

### Breadth Quality
- [ ] Multiple lenses used (not all conventional)
- [ ] Wild ideas included (expand thinking)
- [ ] Combinations explored
- [ ] No obvious category missing

### Depth Quality
- [ ] Ideas are specific, not vague
- [ ] Promising ideas developed further
- [ ] Evaluations are thoughtful
- [ ] Next steps are actionable

### Process Quality
- [ ] Problem properly defined first
- [ ] Constraints clarified
- [ ] Criteria for evaluation appropriate
- [ ] Clustering reveals patterns

### Output Quality
- [ ] Ideas are understandable
- [ ] Recommendations are clear
- [ ] Rationale is provided
- [ ] Action steps are concrete

## Anti-Patterns to Avoid

1. **Premature Criticism**: Evaluating while generating (kills ideas)
2. **Idea Attachment**: Pushing personal favorite regardless of merit
3. **Conformity Bias**: Only generating safe, expected ideas
4. **Analysis Paralysis**: Over-evaluating, under-deciding
5. **Vague Ideas**: "Do marketing better" isn't actionable
6. **Missing Combinations**: Not combining partial solutions
7. **Single Lens**: Only looking at problem one way
8. **Ignoring Wild Ideas**: Missing breakthrough potential

## Special Handling

### For Business/Strategy Challenges
1. Include competitor analysis lens
2. Consider market timing
3. Think about defensibility
4. Include customer perspective
5. Consider second-order effects

### For Creative/Design Challenges
1. Push for more wild ideas
2. Include aesthetic dimensions
3. Consider user experience
4. Think about narrative/story
5. Include inspiration from art/culture

### For Technical Challenges
1. Include build vs buy vs partner
2. Consider scalability
3. Think about maintenance burden
4. Include migration paths
5. Consider technical debt implications

### For Personal Decisions
1. Include values alignment
2. Consider reversibility
3. Think about regret minimization
4. Include "do nothing" option
5. Consider second-order life effects

## Invocation Examples

User: "Brainstorm ways to improve our onboarding"
→ Define success metrics, user segments, then generate across conventional/unconventional/wild

User: "I need creative ideas for my product launch"
→ Understand constraints (budget, timeline), then explore channels, messaging, partnerships

User: "What are some options for my career transition?"
→ Clarify goals and constraints, then explore direct paths, adjacent moves, wild pivots
