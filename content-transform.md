# Content Transformer

Adapt content across formats, audiences, and platforms while preserving core message and maximizing impact.

## Approach

### Phase 1: Source Analysis
Before transforming, fully understand the source:

1. **Content Audit**
   - Core message/thesis: What's the ONE thing?
   - Supporting points: What backs up the core?
   - Evidence/examples: What makes it concrete?
   - Structure: How is it organized?
   - Tone: What's the voice?
   - Length: Word count, sections, depth

2. **Purpose Assessment**
   - Why was this created originally?
   - What was it trying to accomplish?
   - Who was the original audience?
   - What worked well? What didn't?

3. **Asset Extraction**
   - Key quotes/phrases worth preserving
   - Data points/statistics
   - Unique insights/angles
   - Memorable examples
   - Visual opportunities

### Phase 2: Target Definition
Define the transformation goal:

1. **New Format**
   - What format is the target?
   - What are that format's conventions?
   - What constraints exist (length, structure)?
   - What opportunities exist?

2. **New Audience**
   - Who is the new audience?
   - What do they already know?
   - What do they care about?
   - How do they consume this format?

3. **New Context**
   - Where will this appear?
   - What surrounds it?
   - What action should follow?
   - What's the competitive landscape?

### Phase 3: Gap Analysis
Identify what needs to change:

**What to Keep**
- Core message (always)
- Key evidence that translates
- Universal insights
- Strong language/phrases

**What to Adapt**
- Structure (format-dependent)
- Tone (audience-dependent)
- Examples (context-dependent)
- Depth (length-dependent)

**What to Add**
- Format-specific elements
- Audience-relevant context
- Platform conventions
- Calls to action

**What to Remove**
- Irrelevant details
- Audience-inappropriate content
- Length excess
- Outdated references

### Phase 4: Format-Specific Transformation

**Blog Post → Twitter/X Thread**
- Lead with hook (controversial or valuable)
- One idea per tweet
- Add numbers/lists for scanability
- Include engagement hooks
- End with CTA or summary
- Add thread-appropriate formatting

**Report → Executive Summary**
- Lead with conclusion/recommendation
- Include only essential supporting data
- Remove methodology details
- Use bullets over paragraphs
- Add clear next steps

**Long-form → Presentation Slides**
- One concept per slide
- Headlines that make the point
- Minimal text, maximum visual
- Build progressive narrative
- Include transition logic

**Technical → Non-Technical**
- Replace jargon with plain language
- Add analogies for concepts
- Focus on outcomes over mechanisms
- Include "why it matters"
- Remove unnecessary precision

**Formal → Casual**
- Shorter sentences
- Active voice
- Contractions allowed
- Personal pronouns
- Conversational asides

**Article → Email Newsletter**
- Add personal intro
- More conversational
- Clear CTA
- PS for bonus content
- Subject line optimization

### Phase 5: Transformation Execution
Apply changes systematically:

1. Restructure for new format
2. Adjust length to constraints
3. Adapt tone for audience
4. Add format-specific elements
5. Optimize for platform
6. Add appropriate CTA

### Phase 6: Quality Assurance
Verify transformation success:

1. Core message preserved?
2. Format conventions followed?
3. Audience appropriate?
4. Length within constraints?
5. Tone consistent?
6. CTA clear?

## Format Reference Guide

### Social Media Formats
| Platform | Constraints | Best Practices |
|----------|-------------|----------------|
| Twitter/X | 280 char/tweet | Hooks, threads, controversy |
| LinkedIn | ~3000 char | Professional, stories, insights |
| Instagram | Caption ~2200 | Visual-first, hashtags, stories |
| TikTok | Script ~60s | Hook in 3s, entertainment value |

### Professional Formats
| Format | Length | Key Elements |
|--------|--------|--------------|
| Exec Summary | 1 page | Conclusions first, bullets |
| One-Pager | 1 page | Problem-solution-proof |
| Memo | 1-2 pages | BLUF, recommendations |
| Slides | 10-15 | One point/slide, visuals |

### Content Formats
| Format | Length | Key Elements |
|--------|--------|--------------|
| Blog Post | 800-2000 words | Scannable, SEO, value |
| Newsletter | 500-1000 words | Personal, valuable, CTA |
| White Paper | 2000-5000 words | Research, authoritative |
| Case Study | 1000-2000 words | Problem-solution-results |

## Quality Checks

Before delivering transformation, verify:

### Message Integrity
- [ ] Core message preserved accurately
- [ ] No meaning distortion
- [ ] Key evidence retained
- [ ] Attribution maintained where needed

### Format Compliance
- [ ] Length within constraints
- [ ] Structure follows conventions
- [ ] Platform requirements met
- [ ] Format-specific elements included

### Audience Fit
- [ ] Tone appropriate for audience
- [ ] Jargon level correct
- [ ] Examples resonate
- [ ] Assumes correct prior knowledge

### Effectiveness
- [ ] Hooks attention appropriately
- [ ] Flows logically in new format
- [ ] CTA is clear and relevant
- [ ] Would perform well in context

## Anti-Patterns to Avoid

1. **Core Message Drift**: Losing the main point in adaptation
2. **Format Forcing**: Cramming long content into short format
3. **Tone Whiplash**: Inconsistent voice within piece
4. **Over-Simplification**: Losing important nuance
5. **Under-Adaptation**: Just cutting length without restructuring
6. **Platform Ignorance**: Ignoring platform conventions
7. **Audience Mismatch**: Wrong assumptions about reader
8. **CTA Neglect**: No clear next step

## Special Handling

### Highly Technical Content
1. Identify which technical details matter for new audience
2. Create progressive disclosure structure
3. Use analogies liberally
4. Preserve precision where it matters
5. Add "learn more" pathways

### Sensitive/Controversial Content
1. Consider how tone changes perception
2. Be careful with context removal
3. Preserve important nuances
4. Consider platform norms for controversy

### Data-Heavy Content
1. Choose most impactful 2-3 stats
2. Visualize where possible
3. Provide context for numbers
4. Round for readability when appropriate

### Personal/Story Content
1. Keep emotional core
2. Adapt storytelling structure to format
3. Preserve authentic voice
4. Adjust vulnerability level for audience

## Invocation Examples

User: "Turn this blog post into a Twitter thread"
→ Extract hooks, chunk into tweets, add engagement elements, format for threading

User: "Make this technical doc accessible for executives"
→ Lead with business impact, remove jargon, focus on decisions needed

User: "Convert my presentation into a blog post"
→ Expand bullet points, add transitions, include narrative flow, add depth
